数据实体

  • Request
    请求实体： 请求实体包括:
  1. ID，每个请求都有一个唯一的ID作为标识，用于识别与之对应的响应实体，关于ID生成以下会涉及到；
  2. 类型，自定义的类型字段，标明业务类型，whatever…
  3. 服务名，其实就是对应接口(远程方法)所在的类；
  4. 方法名，对应的接口名；
  5. 参数列表，对应接口的参数列表；
  6. 参数类型，参数列表对应的参数类型表。
  • Response
    响应实体： 响应实体包括：
  1. 响应的请求ID；
  2. 响应代码，标识成功或失败等作用；
  3. 错误信息，远程端在执行方法时可以通过抛出异常并写入这个字段来响应到客户端；
  4. 数据实体，即远程方法的返回值。
